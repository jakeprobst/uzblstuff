uzbl setup howto



install
grab and compile the latest tarball off of the uzbl site. package managers are usually either out of date (the uzbl developers like breaking backwards compatiblility regularly) or do odd stuff, so using the source from the main page or git is recommended.

uzbl's parts
uzbl-core
this just shows a webkit view and throws events, useless on its own dont run this.

uzbl-event-manager
this is what handles the events that uzbl-core outputs. uzbl-browser will launch this if it is not already running

uzbl-browser
this is what you want to run, it is a shell script that wraps the calls to uzbl-core, uzbl-event-daemon, uzbl-cookie-daemon and sets some variables.

uzbl-tabbed
tabbed interface for uzbl.

uzbl-cookie-daemon
handles cookies. uzbl-browser will launch this if it is not there

setup
make sure the proper XDG variables are set (XDG_DATA_HOME, XDG_CACHE_HOME, XDG_CONFIG_HOME). these default to ~/.local/share, ~/.cache, ~/.config. uzbl-browser will set these for you, but be aware these exist. on the first run, uzbl-browser should copy the default config from /usr/local/share/uzbl/config to $XDG_CONFIG_HOME/uzbl/.

config file
this is where all your bindings/settings are defined.
important variables
scripts_dir: list of paths where scripts might be located
status_format: how to format the bar along the bottom

bindings
set bind = request BIND
@bind o _ = set uri = %s
the request command just sends a user-defined event to the event manager to handle. this sends request BIND o _ = set uri = %s which tells the event manager to make the command o take a string argument (_), and when enter is hit, set uri to whatever the argument was (o google.com -> set uri = google.com) a list of commands and variables can be found in the README
there is also @cbind, which only works in command mode; @ibind, which only works in insert mode, and @ebind, which works everywhere not insert mode.

events
set on_event = request ON_EVENT
@on_event DOWNLOAD_REQUEST spawn @scripts_dir/download.sh %s \@proxy_url
note that if you want to use a variable that will change in a command, it needs a \ before it, otherwise it will get replaced when the script is loaded.
this just execs a command when a certain event is raised.
